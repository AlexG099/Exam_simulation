В проекте реализована симуляция проведения экзамена с участием нескольких параллельно работающих экзаменаторов. Суть экзамена состоит в последовательном ответе на три вопроса, где студент выбирает случайное слово из вопроса с использованием вероятностного механизма на основе золотого сечения, ответ зависит от пола студента. Экзаменатор заранее не знает правильный ответ и также на основе золотого сечения выбирает несколько слов, считающихся верными, причем каждое следующее слово выбирается с вероятностью 1/3. На результат экзамена также влияет настроение экзаменатора. Время экзамена зависит от длины имени экзаменатора. Каждый экзаменатор работает в отдельном процессе multiprocessing. Студенты помещаются в общую очередь, откуда экзаменаторы забирают их по мере освобождения. Через 30 секунд от начала экзамена каждый экзаменатор имеет право один раз уйти на обед. Для синхронизации данных между процессами используется очередь результатов. Интерфейс реализован с применением библиотеки PrettyTable, обновляется в реальном времени и отображает статус студентов и экзаменаторов. По завершении всех процессов формируется финальная статистика: вычисляется общее количество сдавших и проваливших студентов, процент сдачи; определяются лучшие студенты (сдавшие быстрее всех), лучшие экзаменаторы (с наименьшим процентом заваленных студентов), лучшие вопросы (с наибольшим количеством правильных ответов) и студенты, подлежащие отчислению (провалившие быстрее всех). На основе процента сдавших делается вывод, удался экзамен или нет (более 85% сдавших).
